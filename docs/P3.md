# P3 — Data Plane Bootstrap (Signaling, Discovery, AuthEnvelope)

## Scope

| Component | Deliverable |
|-----------|-------------|
| `@dpe/p2p` | AuthEnvelope validation + first-message handshake; WebRTC signaling message schemas |
| `apps/signaling` | Room-based WebSocket relay (`join` / `leave` / `signal` → `peers` / `signal`) |
| `apps/lan-agent` | mDNS (`bonjour-service`) + manual peers + UID search + discovery WebSocket |

Covers plan **总-2 / 总-3** (network info, LAN neighbors, UID search). Full browser WebRTC mesh and Yjs sync → **P4**.

## Protocol

### Signaling (`ws://host:3002/ws`)

Client → server:

```json
{ "type": "join", "room": "<group_id>", "node_id": "<NodeID>" }
{ "type": "leave", "room": "<group_id>" }
{ "type": "signal", "room": "<group_id>", "to": "<optional NodeID>", "payload": { ... } }
```

Server → client:

```json
{ "type": "peers", "room": "<group_id>", "peers": ["..."] }
{ "type": "signal", "room": "<group_id>", "to": "...", "payload": { "from": "...", ... } }
```

### AuthEnvelope (DataChannel first frame)

After WebRTC connects, the initiator sends `AuthEnvelope` (see `packages/proto`). The responder runs `acceptAuthEnvelope()` from `@dpe/p2p`, which verifies JWT with pinned `pk_admin` and optionally `proof` for the signaling challenge.

### LAN agent

| Method | Path | Description |
|--------|------|-------------|
| GET | `/network` | Hostname, node_id, interfaces, signaling URL |
| GET | `/discovery` | Current peer list |
| GET | `/peers?uid=` | Filter peers by NodeID prefix |
| POST | `/peers/manual` | Register manual peer `{ uid, host, port }` |
| WS | `/ws` | `discovery` events; `ping` → `pong` |

## Env

- `SIGNALING_PORT` (default `3002`)
- `LAN_AGENT_PORT` (default `3003`)
- `DPE_NODE_ID` — this agent's NodeID (UID)
- `DPE_AGENT_NAME` — mDNS display name
- `DPE_MANUAL_PEERS` — `uid@host:port,...` when mDNS unavailable
- `DPE_DISABLE_MDNS=1` — skip Bonjour (CI / restricted networks)
- `DPE_SIGNALING_URL` — advertised to clients
